Python Core Concepts

Comments:

Comments are used to make notes within your code that are ignored by the interpreter. They help in explaining the code's functionality. In Python, comments start with the hash symbol (#). 

# This is a single-line comment

"""
This is a multi-line comment.
You can write multiple lines here.
"""

Variables:

Variables are used to store data. You can assign a value to a variable using the assignment operator (=). 

# Example of assigning a value to a variable
x = 5
name = "John"

Data Types:

Python has several built-in data types, including:

  • Integers: Whole numbers without decimal points (e.g., 5, -2, 0)
  • Floats: Numbers with decimal points (e.g., 3.14, -0.5)
  • Strings: Sequences of characters enclosed in single or double quotes (e.g., "Hello", 'World')
  • Booleans: True or False values representing logical conditions (e.g., True, False)
  • Lists: Ordered collections of items (e.g., [1, 2, 3])
  • Tuples: Immutable ordered collections of items (e.g., (1, 2, 3))
  • Dictionaries: Key-value pairs (e.g., {"name": "John", "age": 25})
# Examples of data types
x = 5  # integer
pi = 3.14  # float
name = "John"  # string
is_student = True  # boolean
my_list = [1, 2, 3]  # list
my_tuple = (4, 5, 6)  # tuple
my_dict = {"name": "John", "age": 25}  # dictionary

Basic Operations:

Python supports various operators for performing operations on variables and values. Some commonly used operators include:

Arithmetic Operators:
  • Addition (+)
  • Subtraction (-)
  • Multiplication (*)
  • Division (/)
  • Modulo (%)
  • Exponentiation (**)
Comparison Operators:
  • Equal to (==)
  • Not equal to (!=)
  • Greater than (>)
  • Less than (<)
  • Greater than or equal to (>=)
  • Less than or equal to (<=)
Logical Operators:
  • And (and)
  • Or (or)
  • Not (not)
Assignment Operators:
  • Assignment (=)
  • Add and assign (+=)
  • Subtract and assign (-=)
  • Multiply and assign (*=)
  • Divide and assign (/=)
# Examples of basic operations

y = 10 - 2  # subtraction

logical_or = True or False  # logical OR

Python "__" and "__init__"

__ (Double Underscore):

In Python, "__" (double underscore) is used for special methods and attributes. These methods have a specific meaning and are invoked by certain language constructs or operations.

Some commonly used special methods include:

  • __init__: The __init__ method is a special method used to initialize objects of a class. It is called automatically when a new instance of the class is created.
  • __str__: The __str__ method is used to represent the class objects as strings. It is called by the built-in str() function and by the print statement.
  • __len__: The __len__ method is used to implement the built-in len() function. It returns the length of the object.
  • __add__: The __add__ method is used to implement the addition operator (+) for objects of a class. It defines the behavior of the "+" operator when applied to the objects.

__init__:

The __init__ method is a special method in Python classes. It is also known as the constructor method. It is automatically called when a new instance (object) of a class is created.

The purpose of the __init__ method is to initialize the attributes (variables) of the object and perform any necessary setup tasks. 

class MyClass:
    def __init__(self, parameter1, parameter2):
        self.attribute1 = parameter1
        self.attribute2 = parameter2

# Creating an instance of the class
obj = MyClass(value1, value2)

Python Dunder Methods (Special Methods)

__init__:

The __init__ method is a constructor method that gets called automatically when an object is created from a class. It is used to initialize the object's attributes. 

class MyClass:
    def __init__(self, parameter):
        self.attribute = parameter

# Creating an object and initializing its attribute
obj = MyClass("Hello")
print(obj.attribute)  # Output: Hello

Other Dunder Methods:

Here are some other commonly used dunder methods along with their explanations and code snippets.

__str__:

The __str__ method is used to return a string representation of the object. It is commonly used for debugging and displaying information about the object. 

class MyClass:
    def __init__(self, parameter):
        self.attribute = parameter

    def __str__(self):
        return f"MyClass object with attribute: {self.attribute}"

obj = MyClass("Hello")
print(obj)  # Output: MyClass object with attribute: Hello
__len__:

The __len__ method is used to define the behavior of the built-in len() function when called on an object. It should return the length of the object. 

class MyList:
    def __init__(self, items):
        self.items = items

    def __len__(self):
        return len(self.items)

my_list = MyList([1, 2, 3, 4, 5])
print(len(my_list))  # Output: 5
__getitem__:

The __getitem__ method is used to define the behavior of accessing elements from the object using square brackets ([]). It allows you to implement indexing and slicing for your custom objects. 

class MyList:
    def __init__(self, items):
        self.items = items

    def __getitem__(self, index):
        return self.items[index]

my_list = MyList([1, 2, 3, 4, 5])
print(my_list[2])  # Output: 3
__main__:

The __main__ dunder method is used to indicate the entry point of a program when the script is run directly. 

def main():
    # Your main program code goes here

if __name__ == "__main__":
    main()

f-strings (Formatted String Literals) and self

f-strings (Formatted String Literals):

In Python, an f-string (formatted string literal) is a way to embed expressions inside string literals, using curly braces ({}) to enclose the expressions. It allows you to easily format and concatenate variables and expressions within strings. 

name = "John"
age = 25
message = f"My name is {name} and I am {age} years old."
print(message)  # Output: My name is John and I am 25 years old.

self:

In Python class methods, including special methods (dunder methods), the first parameter is always self. It represents the instance of the class and is used to access the instance's attributes and methods within the class definition. 

class MyClass:
    def __init__(self, parameter):
        self.attribute = parameter

    def my_method(self):
        print(self.attribute)

obj = MyClass("Hello")
obj.my_method()  # Output: Hello
