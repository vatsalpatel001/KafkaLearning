WSL Installation

wsl --install

This command will enable WSL and install the latest WSL kernel component.

Once WSL is installed, you can install Ubuntu or any other Linux distribution from the Microsoft Store or by using the wsl --install command.

wsl --install -d ubuntu

Understanding "The requested operation requires elevation" Error

The "The requested operation requires elevation" error typically occurs when you try to run a command that requires administrative privileges, but you haven't opened the PowerShell or Command Prompt with administrator rights.

Select "Run as administrator" from the context menu. This will open an elevated PowerShell or Command Prompt instance.

Installing Oracle Java (JDK) on Ubuntu

sudo add-apt-repository ppa:linuxuprising/java

The command sudo add-apt-repository ppa:linuxuprising/java adds the "Linux Uprising" Personal Package Archive (PPA) that provides the Oracle Java (JDK) installer for Ubuntu-based systems. Here's what the command does:

  • sudo: It stands for "Super User DO" and is used to execute the following command with administrative privileges (root access). You'll likely be prompted to enter your password to confirm the action.
  • add-apt-repository: This is a tool in Ubuntu and other Debian-based Linux distributions that allows you to add new software repositories to the system. Repositories are locations that contain software packages.
  • ppa:linuxuprising/java: This is the specific PPA you are adding. PPAs are user-generated repositories that offer additional software or updated versions of existing packages. The PPA "linuxuprising/java" provides the Oracle Java (JDK) installer.

After running this command, Ubuntu will update its package information to include the packages from the newly added PPA. This will allow you to install Oracle Java (JDK) on your system using the package manager. Remember that using third-party PPAs should be done with caution, and you should only add repositories from trusted sources.

Updating Package Information

sudo apt update

When you run sudo apt update, the package manager will contact the software repositories. It checks for updates, new package versions, and any changes to the available software.

Installing Oracle Java 17 JDK

sudo apt install oracle-java17-set-default

This command will attempt to install the Oracle Java 17 JDK and set it as the default Java version on your system, assuming that the package is available in the "Linux Uprising" PPA.

Remember that Oracle Java might have licensing restrictions, and you should review and accept the Oracle JDK license terms during the installation process.

Downloading Apache Kafka

wget https://downloads.apache.org/kafka/3.5.0/kafka_2.13-3.5.0.tgz

The command you provided, wget, is a Linux command-line utility used to download files from the web. However, it is not available by default on Windows systems.

This will download the Apache Kafka 3.5.0 distribution tarball (kafka_2.13-3.5.0.tgz) from the Apache Kafka website.

Extracting Kafka Tarball

tar -zxf kafka_2.13-3.5.0.tgz

The tar command is used to extract files from a tarball (compressed archive). The option -z tells tar to handle gzip-compressed archives, and -x is used to extract the files. The option -f specifies the input file.

Print Working Directory

pwd

The pwd command is a shell command used to print the current working directory. It stands for "print working directory."

Moving Kafka Directory

mv /root/kafka_2.13-3.5.0 /usr/local/kafka

The command you provided, mv, is used to move or rename files and directories in Linux/Unix-based systems. In this case, you are attempting to move the kafka_2.13-3.5.0 from /root to /usr/local/kafka.

Creating Kafka Logs Directory

mkdir /tmp/kafka-logs

Changing Directory

cd /usr/local/kafka/bin

You have successfully changed the current working directory to the /usr/local/kafka/bin directory using the cd command.

Generating Random UUID for Kafka

sudo ./kafka-storage.sh random-uuid

4kRdKzVGT3y0Wlz_MvRuGA

The kafka-storage.sh script is used to generate a random Universally Unique Identifier (UUID) for Kafka. UUIDs are 128-bit numbers used as unique identifiers, and they are commonly used in distributed systems to avoid collisions and ensure data integrity.

Formatting Kafka Storage Log Segment

sudo ./kafka-storage.sh format -t 4kRdKzVGT3y0Wlz_MvRuGA -c /usr/local/kafka/config/kraft/server.properties

By running this command, you are formatting a Kafka storage log segment with the specified UUID and using the provided server properties for configuration.

Starting Kafka Server

sudo ./kafka-server-start.sh /usr/local/kafka/config/kraft/server.properties

The command you provided, sudo ./kafka-server-start.sh /usr/local/kafka/config/kraft/server.properties, is used to start the Kafka server using the specified configuration file server.properties.

Creating Kafka Topic

sudo kafka-topics.sh --create --topic YOUR_TOPIC_NAME --bootstrap-server YOUR_BOOTSTRAP_SERVER --partitions NUM_PARTITIONS --replication-factor REPLICATION_FACTOR

sudo ./kafka-topics.sh --create --topic food --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1

Using Kafka Console Producer and Consumer

sudo ./kafka-console-producer.sh --topic food --bootstrap-server localhost:9092

sudo ./kafka-console-consumer.sh --topic food --bootstrap-server localhost:9092 --from-beginning
