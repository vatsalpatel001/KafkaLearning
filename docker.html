Docker

A platform for running, building, and shipping applications.

Why Docker?

It works on local but may not work on other systems due to different environment setups, system variables, versions, or missing files. Docker comes to the rescue by providing:

  • Consistency
  • Portability
  • Isolated environments using containers

Virtual Machine vs Container

To create new virtual machines, hypervisors like VirtualBox, VMware, and Hyper-V (for Windows) are used. However, virtual machines require a full-blown operating system, are slow to start, and resource-intensive. On the other hand, containers are:

  • More lightweight
  • Use the host's operating system
  • Start quickly
  • Require fewer hardware resources

Docker Client, REST API, and Docker Engine

Docker architecture consists of:

  • Docker client
  • REST API
  • Docker Engine (server)

A container is like a process, and the kernel manages hardware and applications. Linux and Windows have different kernels, so Linux cannot run Windows applications and vice versa.

Getting Started

To get started with Docker:

  1. Download Docker Desktop from https://www.docker.com/products/docker-desktop/
  2. For WSL2 errors, update the Linux kernel. Refer to https://learn.microsoft.com/en-us/windows/wsl/install-manual#step-4---download-the-linux-kernel-update-package
  3. After installation, check the version using docker version

Dockerfile and Images

A Dockerfile is used to create an image, which contains:

  • A cut-down OS
  • A runtime environment
  • Application files
  • Third-party libraries
  • Environment variables

Example Dockerfile:

 FROM node:alpine
 COPY . /app
 CMD node /app/app.js

Creating and Running a Container

To create and run a container from the image:

 $ docker build -t hello-docker .
 $ docker image ls
 $ docker run hello-docker

Once the image is generated, you can access the application from anywhere.
