Explaining Kafka

  • Kafka: Imagine it as a special mailbox for your friends to send messages to each other efficiently.
  • Friends: They can drop their messages into Kafka (the mailbox) instead of sending messages directly to each other.
  • Kafka Topics: Kafka organizes messages into different categories, like topics in the mailbox.
  • Messages: Your friends can talk about different things by putting their messages in different topics.
  • Reading Messages: Everyone can pick up messages from Kafka whenever they want, even if they missed some earlier.
  • Kafka Brokers: These are like additional mailboxes that keep copies of messages to ensure they are safe. If one mailbox stops working, messages are still safe in others.
  • Efficiency: Kafka is super-efficient and perfect for handling large amounts of real-time data reliably.

Kafka Terminology

  • Kafka Cluster: A group of Kafka brokers working together to form a distributed messaging system.
  • Kafka Broker: A server where Kafka runs, responsible for managing storage, handling read and write requests, and replicating messages.
  • Kafka Producer: Writes new data (messages) to the Kafka server for distribution to topics.
  • Kafka Consumer: Reads data from the Kafka server, processing messages from subscribed topics.
  • Zookeeper: Keeps track of Kafka cluster health, manages configuration information, and assists in leader election and coordination.
  • Kafka Connect: Facilitates data integration from external entities like databases or files into Kafka without writing custom code, offering declarative integration.
  • Kafka Streams: Enables data transformation and real-time stream processing within Kafka, a lightweight library for building stream processing applications.

Getting Started with Kafka - Glossary

Term Explanation
Kafka Cluster A group of Kafka brokers working together to form a distributed messaging system.
Kafka Broker A server running Kafka that handles read and write requests, manages storage, and replicates messages.
Kafka Producer A client application that writes data (messages) to Kafka topics for distribution.
Kafka Consumer A client application that reads data from Kafka topics to process the messages.
Topic A logical channel in Kafka where messages are published by producers and consumed by consumers.
Partition A partition is a subset of a topic and represents a sequence of messages.
Offset An offset is a unique identifier of a message within a partition.
Leader The Kafka broker responsible for handling read and write requests for a specific partition.
Replication Replicating data across multiple brokers for fault tolerance and high availability.
Producer API An API provided by Kafka to enable producers to publish data to Kafka topics.
Consumer API An API provided by Kafka to enable consumers to subscribe and receive messages from Kafka topics.
Consumer Group A set of consumers sharing the load of processing messages from a topic.
Zookeeper A distributed coordination service used by Kafka for managing the cluster.
Kafka Connect A framework for integrating Kafka with external systems through pre-built connectors.
Kafka Streams A lightweight library for building real-time stream processing applications on Kafka.
Producer Acknowledgment The mechanism to ensure producers receive acknowledgment from brokers after producing messages.
Consumer Offsets The mechanism that allows consumers to track the last read message in a topic partition.
Retention Policy The policy that determines how long Kafka retains messages in a topic before deleting them.
Message Compression The option to compress messages to reduce network bandwidth and storage utilization.
Partition Rebalance The process of redistributing partitions across brokers to achieve load balancing.

Install Kafka and Zookeeper on Windows

Link to download Kafka binary: https://downloads.apache.org/kafka/3.5.0/kafka_2.13-3.5.0.tgz (100 MB)

.tgz is like a .zip file.

In the unzipped folder, you will find the following directories:

  • bin
  • config
  • libs
  • licenses
  • site-docs

In the "bin/windows" directory, there are bat files that will be run using the command prompt (cmd).

In the "config" directory, some configurations are available, such as zookeeper.properties and server.properties.

Commands to Start Kafka and Zookeeper:

  • Start Zookeeper: zookeeper-server-start.bat ..\..\config\zookeeper.properties
  • Start Kafka: kafka-server-start.bat ..\..\config\server.properties

Other Kafka Commands:

  • Create a topic: kafka-topics.bat --create --topic my-topic --bootstrap-server localhost:9092 --replication-factor 1 --partitions 3
  • Produce messages to a topic: kafka-console-producer.bat --broker-list localhost:9092 --topic my-topic
  • Consume messages from a topic: kafka-console-consumer.bat --bootstrap-server localhost:9092 --topic my-topic --from-beginning

Error and Solutions:

Error 1: The term 'zookeeper-server-start.bat' is not recognized as the name of a cmdlet, function, script file, or operable program. Check the spelling of the name, or if a path was included, verify that the path is correct and try again.

Solution 1: By default, PowerShell restricts the execution of scripts from the current directory to enhance security and prevent accidental execution of malicious scripts. To run the zookeeper-server-start.bat script from the current directory, you should use the .\ prefix to indicate that the script is located in the current directory.

Error 2: The "The input line is too long" error message.

Solution 2: If the paths in the command are too long, try moving the Kafka installation directory to a location with a shorter path or rename some directories to shorten the overall path length.

Error 3: The error message "'java' is not recognized as an internal or external command.

Solution 3: Download and install Java from the official website: https://www.java.com/en/download/
